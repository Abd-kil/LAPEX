# Supabase Database Setup

This guide will help you set up and connect your Next.js application to Supabase PostgreSQL database.

## Prerequisites

- A Supabase account (sign up at https://supabase.com)
- Node.js installed

## Setup Instructions

### 1. Create a Supabase Project

1. Go to https://app.supabase.com
2. Click "New Project"
3. Fill in your project details
4. Wait for the database to be provisioned

### 2. Get Your API Keys

1. Go to Project Settings > API
2. Copy the following values:
   - Project URL
   - `anon` public key
   - `service_role` secret key (for admin operations)

### 3. Configure Environment Variables

Open `.env.local` and replace the placeholder values:

```env
NEXT_PUBLIC_SUPABASE_URL=https://your-project-id.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key-here
SUPABASE_SERVICE_ROLE_KEY=your-service-role-key-here
```

### 4. Run the Database Schema

Execute the SQL schema provided in your database to create the tables:

```sql
-- Create enum types
CREATE TYPE storage_type_enum AS ENUM ('HDD', 'SSD', 'NVME');
CREATE TYPE usage_type_enum AS ENUM ('office', 'gaming', 'editing', 'programming', 'studying');

-- sources
CREATE TABLE sources (
  id INT PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
  name VARCHAR(100) NOT NULL,
  website_url VARCHAR(255) NOT NULL,
  logo_url VARCHAR(255),
  created_at timestamptz NOT NULL DEFAULT now()
);

-- laptops
CREATE TABLE laptops (
  id INT PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
  source_id INT NOT NULL,
  title VARCHAR(255) NOT NULL,
  product_url VARCHAR(500) NOT NULL,
  brand VARCHAR(100),
  model VARCHAR(150),
  price DECIMAL(10,2),
  currency VARCHAR(10) DEFAULT 'SYP',
  is_available BOOLEAN DEFAULT true,
  created_at timestamptz NOT NULL DEFAULT now(),
  updated_at timestamptz NOT NULL DEFAULT now(),
  FOREIGN KEY (source_id) REFERENCES sources(id)
);

-- laptop_images
CREATE TABLE laptop_images (
  id INT PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
  laptop_id INT NOT NULL,
  image_url VARCHAR(500) NOT NULL,
  is_primary BOOLEAN DEFAULT false,
  sort_order INT DEFAULT 0,
  created_at timestamptz NOT NULL DEFAULT now(),
  FOREIGN KEY (laptop_id) REFERENCES laptops(id) ON DELETE CASCADE
);

-- laptop_specs
CREATE TABLE laptop_specs (
  id INT PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
  laptop_id INT UNIQUE,
  cpu VARCHAR(150),
  gpu VARCHAR(150),
  ram_gb INT,
  storage_type storage_type_enum,
  storage_gb INT,
  screen_size DECIMAL(4,1),
  screen_resolution VARCHAR(50),
  refresh_rate INT,
  battery_wh INT,
  weight_kg DECIMAL(4,2),
  os VARCHAR(100),
  FOREIGN KEY (laptop_id) REFERENCES laptops(id) ON DELETE CASCADE
);

-- laptop_scores
CREATE TABLE laptop_scores (
  id INT PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
  laptop_id INT UNIQUE,
  office_score SMALLINT,
  gaming_score SMALLINT,
  editing_score SMALLINT,
  programming_score SMALLINT,
  overall_score SMALLINT,
  FOREIGN KEY (laptop_id) REFERENCES laptops(id) ON DELETE CASCADE
);

-- price_history
CREATE TABLE price_history (
  id INT PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
  laptop_id INT NOT NULL,
  price DECIMAL(10,2),
  recorded_at timestamptz NOT NULL DEFAULT now(),
  FOREIGN KEY (laptop_id) REFERENCES laptops(id) ON DELETE CASCADE
);

-- users
CREATE TABLE users (
  id INT PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
  name VARCHAR(100),
  email VARCHAR(150) UNIQUE,
  created_at timestamptz NOT NULL DEFAULT now()
);

-- user_laptop_evaluations
CREATE TABLE user_laptop_evaluations (
  id INT PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
  user_id INT,
  cpu VARCHAR(150),
  gpu VARCHAR(150),
  ram_gb INT,
  usage_type usage_type_enum,
  calculated_score SMALLINT,
  created_at timestamptz NOT NULL DEFAULT now(),
  FOREIGN KEY (user_id) REFERENCES users(id)
);

-- Trigger function to update updated_at
CREATE OR REPLACE FUNCTION set_updated_at()
RETURNS trigger
LANGUAGE plpgsql
AS $$
BEGIN
  NEW.updated_at = now();
  RETURN NEW;
END;
$$;

-- Trigger for laptops BEFORE UPDATE
CREATE TRIGGER set_updated_at_trigger
BEFORE UPDATE ON laptops
FOR EACH ROW
EXECUTE FUNCTION set_updated_at();
```

### 5. Enable Row Level Security (RLS)

By default, Supabase enables RLS. You need to create policies:

```sql
-- Allow public read access to laptops
ALTER TABLE laptops ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Enable read access for all users" ON laptops FOR SELECT USING (true);

-- Allow public read access to laptop specs
ALTER TABLE laptop_specs ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Enable read access for all users" ON laptop_specs FOR SELECT USING (true);

-- Allow public read access to laptop scores
ALTER TABLE laptop_scores ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Enable read access for all users" ON laptop_scores FOR SELECT USING (true);

-- Allow public read access to laptop images
ALTER TABLE laptop_images ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Enable read access for all users" ON laptop_images FOR SELECT USING (true);

-- Allow public read access to sources
ALTER TABLE sources ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Enable read access for all users" ON sources FOR SELECT USING (true);

-- Allow public read access to price history
ALTER TABLE price_history ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Enable read access for all users" ON price_history FOR SELECT USING (true);
```

## Usage Examples

### Query Laptops

```typescript
import { getLaptops } from '@/app/lib/supabase/queries';

// Get all available laptops
const laptops = await getLaptops({ isAvailable: true, limit: 20 });

// Filter by brand
const dellLaptops = await getLaptops({ brand: 'Dell' });

// Filter by price range
const budgetLaptops = await getLaptops({ maxPrice: 1000 });
```

### Get Laptop Details

```typescript
import { getLaptopWithDetails } from '@/app/lib/supabase/queries';

const laptop = await getLaptopWithDetails(1);
// Returns laptop with specs, scores, images, and source
```

### Use in React Components

```typescript
'use client';

import { useLaptops } from '@/app/lib/supabase/hooks';

function LaptopList() {
  const { laptops, loading, error } = useLaptops({ 
    isAvailable: true, 
    limit: 20 
  });

  if (loading) return <div>Loading...</div>;
  if (error) return <div>Error: {error.message}</div>;

  return (
    <div>
      {laptops.map(laptop => (
        <div key={laptop.id}>{laptop.title}</div>
      ))}
    </div>
  );
}
```

### Admin Operations (Server-side only)

```typescript
import { insertLaptop, upsertLaptopSpecs } from '@/app/lib/supabase/mutations';

// Insert new laptop
const newLaptop = await insertLaptop({
  source_id: 1,
  title: 'Dell XPS 15',
  product_url: 'https://example.com/dell-xps-15',
  brand: 'Dell',
  model: 'XPS 15',
  price: 1499.99,
  currency: 'USD',
  is_available: true,
});

// Add specifications
await upsertLaptopSpecs({
  laptop_id: newLaptop.id,
  cpu: 'Intel Core i7-12700H',
  gpu: 'NVIDIA RTX 3050',
  ram_gb: 16,
  storage_type: 'SSD',
  storage_gb: 512,
  screen_size: 15.6,
  screen_resolution: '1920x1080',
  refresh_rate: 60,
  os: 'Windows 11',
});
```

## File Structure

- `src/app/lib/supabase/client.ts` - Supabase client configuration
- `src/app/lib/supabase/queries.ts` - Read operations
- `src/app/lib/supabase/mutations.ts` - Write operations (admin)
- `src/app/lib/supabase/hooks.ts` - React hooks for real-time data

## Security Notes

- Never commit `.env.local` to version control
- Use `NEXT_PUBLIC_` prefix only for keys that should be exposed to the browser
- Service role key should only be used in server-side code
- Always enable Row Level Security (RLS) in production
- Create appropriate policies for your use case

## Troubleshooting

### Connection Issues

1. Verify your environment variables are correct
2. Check if your Supabase project is active
3. Ensure your IP is not blocked (check Supabase dashboard)

### RLS Errors

If you get "permission denied" errors:
1. Check your RLS policies
2. Ensure policies are created for the correct operations
3. Temporarily disable RLS for debugging (not recommended in production)

### Type Errors

Make sure your database schema matches the TypeScript types in `models.ts`
